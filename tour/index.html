Station 02 · The big picture

From PRD to merged PR — seven altitudes, one workflow.

Agentic Product Ownership treats software as a chain of decisions at increasing resolution. Each Plan altitude has both a Solo skill (formulate the artifact) and a Meet skill (Council decomposes it into the next altitude down). Build is autonomous; Deliver is human-only. You drop in at whichever altitude your project actually needs.

Station 03 · The team — one plugin, one standard

Install once per person. Standardize the whole team.

The plugin ships through the Claude Code marketplace — every developer, every product owner, every tech lead installs the same toolkit with one command. The repo carries the shared config in .weside/, version-controlled like code. Result: one vocabulary, one plan format, one quality bar — regardless of who (or which Companion) is typing.

Station 05 · Crew & Council

Crew & Council — your lenses, real or generic.

Every Council and Meeting convenes a roster of role-lenses — Architect, Product Owner, Security, UX, and more. /we:onboarding builds that council from scratch, even with zero Companions and no account. For each role you pick one of three: assign an existing Companion, seed a fresh one with the role-lens, or take the shipped generic lens (a „Retorte"). A council where some lenses are real Companions and others are generic is the normal case — mixed by design.

The loadCouncilFromWeside toggle. At convene time this boolean (default true) decides the source: true uses your weside-backed Companions wherever the .weside/council.json bridge links them; false always convenes the generic role-lenses (Retorte), even if Companions exist. Either way the council convenes — an account upgrades the lenses with identity, voice, and memory; it is never a prerequisite.

Station 06 · Vision · PRD altitude

/we:vision + /we:meet vision — why does this product exist?

Station 07 · Saga · Theme altitude

/we:saga + /we:meet saga — the year-shaping bet.

A Saga is a multi-bet theme inside the Vision — something big enough to need decomposition, small enough to name the win. „Make the platform multi-tenant." „Launch the marketplace." Solo (/we:saga) runs Status by default (mirror child Epics from ticketing, snapshot + drift + risk-driven next move) and tightens wording on explicit refine intent; Meet (/we:meet saga) validates the bet against the PRD and decomposes it into Epics.

Station 10 · Build · the autonomous pipeline

/we:build — hand it a ticket, watch it ship.

Nine steps from worktree to PR. Sequential where order matters, parallel where it doesn't. Checkpoints between every step — interruption-safe. By the time you handed it a ticket, you already decided to build it; /we:build doesn't ask permission for each phase.

01

Git prep

Worktree, branch, ticket → In Progress.

02

Develop

Implement plan phase by phase. TDD where it fits. Auto-fix after each phase.

03
Blocking

AC + DoD verify

Every AC checked with concrete evidence — no item passes without a file:line citation — plus the DoD checked against the diff.

04

Simplify

Remove dead code, simplify expressions, reuse helpers.

05
Parallel

Quality gates

One writer-aware reviewer + static + tests, in a single message — concurrent.

06

Docs

doc-architect proposes diffs. Never writes autonomously.

07

PR

Verifies all three quality gates first. Won't create a PR with failing gates.

08
Loop

CI fix

Collect ALL findings, fix in ONE commit, push ONCE. One pass by default (max 2 cycles when looping).

09

Hand off

Ticket → In Review. Never „Done" — that move stays with you.

AC + DoD · Blocking
No claim without citation
Step 3 won't pass an AC without file:line evidence, and checks the DoD against the diff. Halts and asks if it can't.
Gates · Parallel
One reviewer, one wait
Step 5 dispatches review + static + tests in a single message. One writer-aware reviewer — the model that didn't write the code.
CI Loop · Bounded
Batch-fix, one pass
Findings collected, fixed in one commit, pushed once. One pass by default; if it loops, stop & ask after 2 cycles.
Resume · Safe
Checkpoint + circuit breaker
SQLite checkpoints survive interruption. 3 same-phase failures trip the breaker.

/we:build is the solo fast path shown above — one Story, one pipeline, one PR. For multi-chunk or phased work use /we:orchestrate: the Lead dispatches workers via /we:develop (implement → local gates → commit → push, no PR, no CI). Workers run on cheap Claude (Sonnet/Haiku) by default — no extra install. Two optional backends: Codex (gpt-5-codex, opt-in) and foreign engines (any Anthropic-compatible endpoint via .weside/engines.local.json). Workers push branches; the Lead merges them onto one integration branch and runs CI once. Cross-review is on by default: whichever engine wrote the code, the other reviews it — Claude wrote → /codex:adversarial-review; Codex or a foreign engine wrote → the Claude code-reviewer. The plugin never hard-depends on Codex or any foreign engine.

Station 11 · Coach · the APO advisor

/we:coach — where am I, what's next?

One job: orient you. The Coach reads repo state — your plans, recent commits, open tickets, pipeline checkpoints — maps it to an APO altitude, and proposes the natural next /we:* command with a [y/n] gate. Companion-aware: with weside, the Coach speaks in your Companion's voice. For systematic retrospectives after Deliver, see Station 12 — /we:retro owns that loop.

B · BEGINNER

First-Use Orientation

No .weside/ yet? The Coach offers to run /we:setup right now. Set up but no plans? It points at /we:story "your first feature". The on-ramp for new repos and new users.

→
A · ADVISOR

„where am I, what's next?"

Reads docs/plans/, recent commits, open tickets, pipeline state. Maps to altitude (Vision / Saga / Epic / Story / Build), proposes the natural next move with [y/n]. Mid-Saga or mid-Epic: surfaces a one-line Plan-status ("Epic: Ledger — 6 Stories, 2 shipped, 1 active, 3 refined") and points you to /we:saga or /we:epic for the full Status dashboard. Build is always handed off, never launched from a Coach session.

→
CONFIRMATION

You decide. Always.

Every advisor-launched command waits for [y/n]. The Coach proposes; you decide — that's the discipline. Doc changes are delegated to /we:docs; retros are handed to /we:retro; the Coach itself stays in its advisory lane.

Station 12 · Retro · the harness improves itself

/we:retro — every error happens exactly once.

After Deliver, the loop closes. /we:retro walks the shipped PR — session transcript + gh api (CI, review-bot threads, push-fix cycles) — and proposes concrete MD-file edits, primarily in your repo's .claude/rules/ and CLAUDE.md. Per-item [y/n] gate. The harness grows tighter every cycle.

Station 13 · Deliver — you

What you get. What stays yours.

By the time /we:build hands off, the PR is in a known-good state. The remaining work — judging it, merging it, closing the ticket — is human work. By design.

What /we:build produces

  • All acceptance criteria implemented and evidenced
  • Tests passing (unit + relevant integration)
  • Code reviewed (Codex adversarial-review locally + Claude Review on GitHub)
  • Doc diffs proposed and applied
  • CI green (all review threads resolved)
  • Ticket sitting in „In Review"

What stays yours

  • Reading the PR with your eyes
  • Judging the change — is this still what we wanted?
  • Merging the PR
  • Closing the ticket
  • Communicating the change to stakeholders
  • The decision to ship
Claude never merges PRs or closes tickets. Not a feature flag, not a configuration — a property of the design. Autonomy ends at the gate where consequence begins.
Station 14 · Where this goes

Four maturity levels. Start free, grow when it pinches.

Agentic Product Ownership meets you where you are. The plugin alone carries a team to Level 2; the weside platform lifts the same skills — same names, same pipeline — to proactive and organisational maturity. No level is forced; you upgrade when the level below stops fitting.
