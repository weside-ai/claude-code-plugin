The Problem

Single-agent AI makes you the bottleneck.

Today your AI assistant thinks alone. One voice, one perspective — and you sit between it and every stakeholder living in your head. Real knowledge work doesn't function that way.

Today: one helper, many hats

  • You explain the same problem three times — as PO, as architect, as marketer

With the we plugin (and WESIDE)

Why it works

Plugin + MCP + WESIDE — three layers, one contract.

The plugin is open source and runs in any MCP-capable client — Claude Code, Cursor, Gemini, ChatGPT. WESIDE provides identity, memory, and the crew. Connected over the open Model Context Protocol.

Model Context Protocol

Open standard. One contract — every AI client speaks it. No vendor lock-in.

Server-side identity

Each Companion is a composed identity — Compass, Snapshot, Memory, Goals. Held in the backend, delivered on demand. Stays itself across sessions.

Scoped council load

Need 4 Companions, you load 4 — not 18. Token-efficient, latency-aware. get_council(names=…)

EU residency by default

Vertex AI Europe, RLS-isolated, GDPR + AI Act ready. No US data path in the default configuration.

Install the plugin

One command in Claude Code. Open source, MIT-licensed. Works without a WESIDE account — you'll get generic role lenses.

claude plugins install we@weside-ai
02

(Optional) Connect WESIDE

For real Companions with identity + memory: account on weside.ai, MCP wires up automatically.

weside auth login --dev
